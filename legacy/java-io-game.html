📥 1. 字节流体系

核心概念：以字节（8位）为单位进行读写，适合处理二进制数据（图片、视频、文件等）。

InputStream（输入流基类）：

  • FileInputStream - 文件输入流
  • BufferedInputStream - 缓冲输入流，提高效率
  • ObjectInputStream - 对象反序列化

OutputStream（输出流基类）：

  • FileOutputStream - 文件输出流
  • BufferedOutputStream - 缓冲输出流
  • ObjectOutputStream - 对象序列化

📝 2. 字符流体系

核心概念：以字符（16位Unicode）为单位进行读写，适合处理文本数据，支持中文等多语言。

Reader（输入流基类）：

  • FileReader - 文件字符输入流
  • BufferedReader - 缓冲字符输入流，提供 readLine() 方法
  • InputStreamReader - 字节流转字符流的桥梁

Writer（输出流基类）：

  • FileWriter - 文件字符输出流
  • BufferedWriter - 缓冲字符输出流
  • OutputStreamWriter - 字符流转字节流的桥梁

⚡ 3. 缓冲流

核心概念：使用内部缓冲区减少对底层流的直接 I/O 操作，大幅提高读写效率。

关键知识点：

  • 缓冲区大小通常为 8KB
  • 缓冲区满时自动刷新（flush）到磁盘
  • 手动调用 flush() 可强制刷新缓冲区
  • newLine() 方法可跨平台写入换行符

📁 4. 文件操作

File 类：表示文件或目录的抽象路径，可进行文件操作（创建、删除、查询等）。

try-with-resources：

  • Java 7+ 引入的语法
  • 自动关闭实现了 AutoCloseable 接口的资源
  • 即使发生异常也能正确关闭资源

常见异常：

  • FileNotFoundException - 文件不存在
  • IOException - I/O 操作失败

🔮 5. NIO (New I/O)

核心组件：

  • Buffer（缓冲区） - 数据的临时存储容器
  • Channel（通道） - 数据的读写通道，支持双向操作
  • Selector（选择器） - 多路复用，适合高并发网络编程

Buffer 关键方法：

  • flip() - 切换到读模式（limit=position, position=0）
  • clear() - 清空缓冲区（切换到写模式）
  • compact() - 压缩，保留未读数据

💡 6. 最佳实践

选择合适的流：

  • 二进制文件 → 字节流
  • 文本文件（含中文）→ 字符流
  • 大文件 → 缓冲流 + 合理缓冲区大小

资源管理：

  • 始终使用 try-with-resources
  • 不要忘记关闭流

编码处理：

  • 注意字符编码（UTF-8、GBK等）
  • 使用 InputStreamReader/OutputStreamWriter 指定编码
